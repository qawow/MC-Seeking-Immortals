## Phase 0：前置准备与架构审查

**目标：** 审查现有代码结构，确保与设计文档兼容，准备开发环境

2. 检查现有 `Realm` 和 `RealmStage` 是否支持13层练气 + 筑基
3. 检查现有 `SpiritualRoot` 系统是否支持六种灵根类型及其修炼速度系数
4. 审查 `CultivationHelper` 和 `CultivationProvider` 是否适配 MVP 需求
5. 确认 `skill/` 和 `technique/` 系统当前状态
6. 确认 `item/pill/` 系统是否支持品质分级
7. 列出需要新增的类/接口清单
8. 列出需要重构的类/接口清单

### 验收标准

- [ ] 生成架构审查报告：`project_docs/phase0_architecture_review.md`
- [ ] 明确列出与设计文档的差异点
- [ ] 列出需要新增的模块清单

### 禁止事项

- ❌ 不修改任何 Java 代码
- ❌ 不添加新的类或包
- ❌ 不运行构建或测试

## Phase 1：核心属性与境界系统

**目标：** 实现六大核心属性存储与境界系统，支持练气1~13层和筑基4阶

### 任务

1. 确保 `PlayerCultivation` 存储以下字段：
   - `cultivation` (long), `cultivationMax` (long)
   - `mana` (int), `manaMax` (int)
   - `divSense` (int)
   - `bodyRef` (int)
   - `qiDevRisk` (float)
   - `tribRes` (float, 预留)
2. 实现或验证 `RealmStage` 枚举，包含13层练气 + 筑基初/中/后/圆满
3. 为每个境界配置属性基准表（参考设计文档 §1.3）
4. 实现运行时计算的衍生属性（攻击力、防御力、最大HP、灵力回速、修为回速、飞行速度）
5. 添加境界属性基准表单元测试

### 验收标准

- [ ] `PlayerCultivation` 包含所有六大核心属性字段
- [ ] `RealmStage` 枚举至少包含 QI_1 到 QI_13 和 FOUNDATION_EARLY/MID/LATE/PEAK
- [ ] 每个境界有正确的 `manaBase`, `divSenseBase`, `hpBase` 值
- [ ] 衍生属性计算方法实现并通过单元测试
- [ ] `/seeking_immortals realm` 命令能正确显示当前境界和属性

### 禁止事项

- ❌ 不实现突破逻辑（留给 Phase 3）
- ❌ 不实现打坐 GUI（留给 Phase 2）
- ❌ 不实现技能系统（留给 Phase 4）
- ❌ 不添加金丹及以上境界的完整实现（仅预留枚举）

## Phase 2：灵根系统与打坐修炼

**目标：** 实现灵根测试、永久写入、打坐修炼 GUI 和修为增长逻辑

### 任务

1. 实现或验证 `SpiritRootType` 枚举：天/异/双/三/伪/杂灵根
2. 为每种灵根配置修炼速度系数、灵力回复系数、突破成功率加成、丹药吸收率（参考 §1.4）
3. 实现灵根测试触发机制：
   - 创建「灵根鉴定石板」方块/物品
   - 首次交互触发灵根测试
   - 随机分配灵根（伪/杂灵根概率较高，天灵根极低）
   - 永久写入 `PlayerCultivation`
4. 创建打坐修炼 GUI（`MeditationScreen`）：
   - 显示当前修为 / 修为上限
   - 显示修炼速度（基准 × 灵根系数 × 打坐加成 × 灵脉加成）
   - 显示实时修为增长
5. 实现打坐修炼逻辑（`PlayerTickEvent`）：
   - 玩家进入打坐状态（静止）
   - 每 tick 增加修为：`(境界基准 × 灵根系数 × 打坐加成 × 灵脉加成) / 20`
   - 打坐时受伤：打断打坐，走火风险 +2%
   - 打坐消耗饥饿度
6. 实现或验证现有 `MeditationCushionBlock` 是否支持打坐状态

### 验收标准

- [ ] 灵根鉴定石板方块/物品可交互
- [ ] 玩家首次测试灵根后，数据永久写入
- [ ] `/seeking_immortals root` 命令显示灵根类型和系数
- [ ] 打坐 GUI 正确显示修为增长速度
- [ ] 打坐时修为实时增长
- [ ] 打坐时受伤会打断并增加走火风险
- [ ] 不同灵根的修炼速度差异明显（天灵根 ×5.0，杂灵根 ×0.8）

### 禁止事项

- ❌ 不实现神秘小瓶（留给 Phase 5）
- ❌ 不实现灵脉/灵地加成的完整逻辑（可预留接口）
- ❌ 不实现 NPC 墨老先生对话（留给 Phase 8）

## Phase 3：突破系统与走火入魔

**目标：** 实现境界突破流程、成功率计算、走火入魔四级判定

### 任务

1. 实现突破触发机制：
   - 当 `cultivation >= cultivationMax` 时，允许玩家触发突破
   - 突破按键绑定或 GUI 按钮
2. 实现突破资源检查：
   - 检查玩家背包是否有对应丹药（凝气丹、筑基丹等）
   - 显示所需材料和当前拥有量
3. 实现突破成功率计算：
   - 基础成功率（练气各层90%，练气13→筑基60%）
   - 灵根加成（天灵根+25%，杂灵根-18%）
   - 辅助丹药加成（+5% ~ +20%）
   - 地点加成（地灵之眼+15%，Phase 3可暂不实现地点）
   - 连续失败积累「执念」（每次失败+5%，最多+30%）
4. 实现突破结果：
   - 成功：境界+1，修为清零，属性加成，消耗材料
   - 失败：修为回退20%，走火风险+10%，消耗材料
5. 实现走火入魔机制：
   - 触发条件：走火风险 ≥70% 时突破失败、修炼时受伤、使用超阶功法
   - 四级判定表（70-79% / 80-89% / 90-99% / 100%）
   - 效果实现：损失修为、昏迷（debuff）、掉境界、死亡+掉落
6. 实现走火风险降低方式：
   - 服用稳神丹：-20%
   - 平稳打坐（不使用技能）：每小时-5%（ServerTickEvent）

### 验收标准

- [ ] 修为达到上限后，突破按钮/按键可用
- [ ] 突破前显示成功率和所需材料
- [ ] 突破成功后境界正确+1，修为清零
- [ ] 突破失败后修为-20%，走火风险+10%
- [ ] 走火风险≥70%时突破失败触发走火判定
- [ ] 四级走火效果正确实现（修为损失/昏迷/掉境界/死亡）
- [ ] 稳神丹正确降低走火风险20%
- [ ] 平稳打坐每小时降低走火风险5%
- [ ] 连续失败积累执念机制正常工作

### 禁止事项

- ❌ 不实现地灵之眼地点加成（留给后续 Phase）
- ❌ 不实现金丹品阶系统（留给阶段二）
- ❌ 不实现天劫系统（留给阶段二/三）

## Phase 4：练气期技能系统

**目标：** 实现练气期9个技能，包括技能解锁、冷却、消耗、效果

### 任务

1. 实现或验证技能注册系统（`SkillType` 或 technique system）
2. 实现9个练气期技能：
   - **引气入体（被动）**：练气1，解锁打坐修炼
   - **火球术**：练气3，10灵力，2s冷却，发射火球弹射物
   - **冰锥术**：练气3，10灵力，2s冷却，发射冰锥+减速2s
   - **雷击术**：练气3，12灵力，3s冷却，召唤闪电（`Level#addFreshEntity(LightningBolt)`）
   - **土遁步**：练气4，15灵力，5s冷却，向前瞬移穿过1格方块
   - **御剑飞行（初）**：练气7，5灵力/s，骑乘飞剑实体，速度×1.2
   - **单剑刺击**：练气7，20灵力，1s冷却，发射飞剑弹射物攻击
   - **灵气探测**：练气10，5灵力，10s冷却，神识范围内高亮灵矿/灵草
   - **三才剑阵**：练气13，40灵力，3s冷却，三把飞剑同时攻击
3. 实现技能解锁逻辑：境界达到要求后自动解锁
4. 实现技能冷却系统（`PlayerCultivation` 存储 `skillCooldowns[]`）
5. 实现灵力消耗检查
6. 实现飞剑相关技能的自定义实体：
   - `FlyingSwordEntity` 用于御剑飞行（可骑乘）
   - `SwordProjectileEntity` 用于单剑刺击/三才剑阵（弹射物）

### 验收标准

- [ ] 练气1后自动解锁引气入体
- [ ] 练气3后解锁火球术/冰锥术/雷击术
- [ ] 火球术/冰锥术发射弹射物，命中造成伤害
- [ ] 雷击术在目标位置召唤闪电
- [ ] 土遁步正确瞬移玩家并穿过1格方块
- [ ] 练气7后解锁御剑飞行，玩家可骑乘飞剑移动
- [ ] 单剑刺击发射飞剑攻击，射程=神识范围
- [ ] 练气10后灵气探测高亮灵矿/灵草（发光轮廓）
- [ ] 练气13后三才剑阵发射三把飞剑
- [ ] 技能冷却正确倒计时
- [ ] 灵力不足时技能无法释放

### 禁止事项

- ❌ 不实现筑基期技能（留给 Phase 6）
- ❌ 不实现伤害计算公式的完整版（可使用简化版）
- ❌ 不实现技能槽位系统（可直接解锁）

## Phase 5：神秘小瓶系统

**目标：** 实现神秘小瓶唯一物品、灵液积累、植物加速

### 任务

1. 创建 `MysteriousVialItem` 类：
   - 唯一绑定物品，不可丢弃（`onDroppedByPlayer` 返回 false）
   - NBT 存储 `vialCharges` (当前灵液份数) 和 `vialLastRefill` (上次充能时间戳)
2. 实现灵液积累逻辑（`PlayerTickEvent` 或 `PlayerLoggedInEvent`）：
   - 每现实24小时积累1份灵液
   - 最大5份
   - 离线时间也计算（`(currentTime - vialLastRefill) / 86400000`）
3. 实现植物加速使用：
   - 右键对植物/灵草方块使用
   - 消耗1份灵液
   - 植物生长速度×10（蓝色灵液，初期）
   - 实现方式：`BlockGrowEvent` 监听，加速作物成熟
4. 实现灵根与神秘小瓶获取关联：
   - 伪灵根/杂灵根测试后必定获得
   - 其他灵根极低概率获得（或通过任务获取）
5. 添加神秘小瓶物品模型、材质、本地化

### 验收标准

- [ ] 伪灵根/杂灵根玩家测试灵根后自动获得神秘小瓶
- [ ] 神秘小瓶无法丢弃
- [ ] 每24小时自动积累1份灵液（最大5份）
- [ ] 离线时间正确计算并补偿灵液
- [ ] 右键对作物使用消耗1份灵液
- [ ] 使用后作物生长速度明显加快（×10）
- [ ] 神秘小瓶 tooltip 显示当前灵液份数

### 禁止事项

- ❌ 不实现绿色/金色/彩虹灵液升级（留给阶段二/三）
- ❌ 不实现灵液注入丹炉/炼器台功能（留给阶段二）
- ❌ 不实现天雷竹特殊交互（留给阶段二）

## Phase 7：炼丹系统

**目标：** 实现丹炉方块、丹方系统、炼丹技能、核心丹药

### 任务

1. 创建或验证 `AlchemyFurnaceBlock` 和对应的 `BlockEntity`
2. 实现丹方系统（Recipe Manager）：
   - 自定义 Recipe 类型 `AlchemyRecipe`
   - JSON 配方格式：`data/seeking_immortals/recipes/alchemy/`
3. 实现炼丹流程：
   - 放入丹方 + 灵草材料
   - 催火（消耗灵力或灵火石）
   - 等待时间（Progress bar）
   - 判定成功/失败（技能等级 + 材料新鲜度 + 灵地加成）
4. 实现品质判定：
   - 下品（基础）
   - 中品（技能等级高时概率出现）
   - 上品（技能等级高+材料新鲜时概率出现）
   - 失败：产出废丹
   - 小概率爆炉（丹炉损耗）
5. 实现炼丹技能等级系统：
   - LV1~LV10
   - 炼丹成功时获得经验
   - 等级提升解锁更高阶丹药配方
6. 实现核心丹药物品：
   - **凝气丹（一阶）**：服用后1小时修为×2
   - **筑基丹（二阶）**：练气→筑基突破必需（极稀缺）
   - **稳神丹（一阶）**：走火风险-20%
   - **回灵丹（二阶）**：立刻恢复50%灵力
7. 添加对应的灵草材料物品（浮云草、碧云草、万年灵草、血灵芝、天灵草、定神草、回灵草、玉露）

### 验收标准

- [ ] 丹炉方块可放置和交互
- [ ] 丹炉 GUI 显示配方、材料槽、进度条
- [ ] 放入正确材料后可开始炼丹
- [ ] 炼丹成功后产出对应品质的丹药
- [ ] 炼丹失败产出废丹或爆炉
- [ ] 炼丹技能等级正确积累
- [ ] 凝气丹服用后修炼速度×2，持续1小时
- [ ] 筑基丹可用于练气→筑基突破
- [ ] 稳神丹正确降低走火风险20%
- [ ] 回灵丹立刻恢复50%灵力
- [ ] 所有丹药有对应材料配方

### 禁止事项

- ❌ 不实现四阶以上丹药（留给阶段二/三）

## Phase 9：七玄门任务线

**目标：** 实现七玄门完整5阶段任务线

### 任务

1. 创建任务系统基础架构：
   - `Quest` 类：questId, stages, rewards
   - `QuestProgress` 存储于 `PlayerCultivation`
   - 任务阶段追踪
2. 实现七玄门 arc 5阶段：
   - **阶段一：测灵根**
     - 触发：首次与墨老先生对话
     - 任务：参加村镇集合 → 灵根测试 → 被宗门收入
     - 完成条件：灵根测试完成
   - **阶段二：七玄门入门**
     - 触发：测灵根完成
     - 任务：领取黄龙功 → 完成劳役任务（收集材料×10）→ 学习基础炼丹
     - 完成条件：炼丹技能达到 LV1
   - **阶段三：发现秘密**
     - 触发：入门完成
     - 任务：偷入墨老密室（特定坐标）→ 发现神秘小瓶 → 实验使用
     - 完成条件：使用神秘小瓶1次
   - **阶段四：宗门内斗**
     - 触发：发现秘密完成
     - 任务：调查长老勾结 → 收集证据（找到特定物品）→ 三岔路口选择（举报/沉默/勒索）
     - 完成条件：做出选择
   - **阶段五：离开大燕**
     - 触发：宗门内斗完成
     - 任务：天罡盟攻打事件 → 逃离 → 前往越国传送门
     - 完成条件：传送到越国
3. 实现任务奖励发放：
   - 神秘小瓶（如果尚未获得）
   - 黄龙功功法书
   - 下品灵石×300
4. 添加任务追踪 GUI（可选）
5. 实现任务相关的结构生成（墨老密室、传送门等）

- [ ] 阶段五完成后玩家传送到越国
- [ ] 任务奖励正确发放
- [ ] `/seeking_immortals quest` 命令显示当前任务进度

## Phase 10：越国六大宗门任务线（前3阶段）

**目标：** 实现越国六大宗门选择、叩门考核、筑基困局、内门竞争

### 任务

1. 实现六大宗门数据结构：
   - 黄枫谷、天星宗、落月宗、女灵派、凌霄剑派、雷神斋
   - 每个宗门：收徒灵根线、专长、散修态度
2. 实现越国任务线前3阶段：
   - **阶段一：叩门考核**
     - 触发：抵达越国后与招募 NPC 对话
     - 任务：根据灵根选择可加入的宗门 → 完成三关考核 → 成为外门弟子
     - 三关考核：战斗测试、采集测试、知识问答
     - 完成条件：通过考核，获得外门弟子身份
   - **阶段二：筑基困局**
     - 触发：成为外门弟子后
     - 任务：收集筑基丹材料（万年灵草×1 + 血灵芝×2 + 天灵草×1）→ 使用神秘小瓶加速万年灵草生长 → 完成宗门任务积分
     - 完成条件：获得筑基丹材料，积分达到500
   - **阶段三：内门竞争**
     - 触发：筑基困局完成且突破到筑基期
     - 任务：参加年度内门选拔 → 与单灵根 NPC 竞争 → 完成叛徒调查支线
     - 完成条件：选拔通过，成为内门弟子
3. 实现宗门贡献积分系统：
   - 完成宗门任务获得积分
   - 积分可兑换资源/功法
4. 实现宗门任务生成器（简单版）：
   - 采集任务：收集材料×N
   - 击杀任务：击杀妖兽×N
   - 护送任务：护送 NPC 到目标地点
5. 实现叛徒调查支线（分支选择）：
   - 举报：正道声望+50
   - 利用：获得特殊材料
   - 沉默：无影响
6. 实现六大宗门建筑结构生成（简化版）

### 验收标准

- [ ] 越国六大宗门 NPC 可交互
- [ ] 玩家可根据灵根选择并加入宗门
- [ ] 叩门考核三关正确实现
- [ ] 筑基困局任务正确引导玩家使用神秘小瓶
- [ ] 宗门贡献积分正确积累
- [ ] 内门选拔竞争正确判定
- [ ] 叛徒调查支线分支选择生效
- [ ] 完成前3阶段后获得内门弟子身份

### 禁止事项

- ❌ 不实现阶段四「宗门外战」和阶段五「真传传功」（留给阶段二）
- ❌ 不实现南海 arc（留给阶段二）
- ❌ 不实现天南 arc（留给阶段二）

## 执行规则

1. **严格顺序执行**：必须按 Phase 0 → Phase 1 → ... → Phase 10 顺序执行
2. **验收后才能进入下一阶段**：每个 Phase 的所有验收标准必须通过
3. **禁止跨 Phase 开发**：不允许在当前 Phase 中实现后续 Phase 的内容
4. **每个 Phase 完成后**：
   - 运行 `./gradlew build` 确保编译通过
   - 运行相关测试（如有）
   - 更新 `project_docs/step_progress.md`
   - 添加 update note 到 `project_docs/updates/`
5. **如果某个 Phase 阻塞**：
   - 记录阻塞原因
   - 寻找最小可行替代方案
   - 不允许跳过 Phase
