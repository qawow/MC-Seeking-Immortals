# 20260611 0.1.32：移除旧第三方 UI 兼容并重实现原生 UI

## 主要变更

- 移除 Gradle 中旧第三方 UI 仓库与依赖。
- 移除 `mods.toml` 中旧 UI 库 mandatory 依赖声明。
- 删除旧兼容皮肤类命名，改为 `ImmortalUiSkin`。
- `ImmortalUiSkin` 只使用原生 `GuiGraphics.fill`、`blit`、`drawString`、`ResourceLocation` 等 Minecraft 客户端 API。
- 左侧 7 槽技能栏保持 Forge Overlay 实现，继续靠左显示，保留 tooltip、编号与技能占位图标。
- 打坐吐纳 HUD 保持 Forge Overlay 实现，继续使用 `cultivation_progress_bar.png` 显示 5 秒结算进度条。
- 修仙面板保持 B 方案独立 `Screen`，保留基础、灵根、功法、负面状态四分页。
- 背包“修仙”按钮只作为入口打开独立 Screen，不在背包界面叠画完整修仙面板。
- HUD 在任意 Screen 打开时继续隐藏，避免遮挡背包和修仙面板。

## 客户端/服务端隔离

- UI 类仍位于 `client` 包。
- Overlay、按键与背包按钮注册仍通过 `Dist.CLIENT` 客户端事件隔离。
- 通用/服务端逻辑未直接引用客户端 UI 类。

## 验证

- 全局 grep 排除 `.gradle`、`build`、`backups` 后，未发现旧 UI 库关键词或旧皮肤类名残留。
- `./gradlew build` 成功。
- 构建产物：`build/libs/seeking_immortals-0.1.32.jar`。
