# 20260609 0.1.26：修复背包物品栏显示并完善 旧第三方 UI 风格修仙 UI

## 版本

- `gradle.properties` 中 `mod_version` 已更新为 `0.1.26`。
- 构建产物：`build/libs/seeking_immortals-0.1.26.jar`。

## Bug 修复

- 修复打开背包“修仙”页后玩家物品栏不显示物品的问题。
- 根因：上一版 `CultivationStatsScreen` 继承普通 `Screen`，点击“修仙”页签后直接替换原版 `InventoryScreen`，因此不会渲染 `InventoryMenu` 的原版槽位和物品。
- 修复：`CultivationStatsScreen` 改为继承 `InventoryScreen`，打开时传入当前 `LocalPlayer`，先执行原版背包渲染，再在右侧追加修仙信息面板。

## UI 显示复修

- 复查后继续修复“HUD/物品显示依赖先打开背包初始化”和渲染状态污染风险。
- `TechniqueSkillBarOverlay` 与 `BreathingHudOverlay` 不再订阅任意 `RenderGuiOverlayEvent.Post`，改为由 `ClientEvents.registerGuiOverlays` 在 MOD bus 通过 `RegisterGuiOverlaysEvent.registerAboveAll` 注册，生命周期更接近 Forge/旧第三方 UI 的 Screen/组件/Overlay 分层思路。
- 技能栏与吐纳 HUD 在 `minecraft.screen != null` 时不再渲染，避免覆盖背包/修仙页或污染物品栏渲染状态。
- `CultivationStatsScreen` 的 旧第三方 UI 风格修仙信息面板移入 `renderBg`，由 `InventoryScreen` 在其后继续渲染原版 slot、item、按钮与 tooltip，确保玩家物品栏不依赖先打开背包初始化。

## 旧第三方 UI 风格 UI 迁移

- 背包“修仙”页改为兼容式 旧第三方 UI 风格分页：
  - 基础状态
  - 灵根信息
  - 功法信息
  - 负面状态
- 功法/灵根信息已经拆为独立渲染方法，作为后续点击技能槽进入详情页的入口准备。
- 仍沿用 `ImmortalUiSkin` 适配层，只使用已验证可编译的 旧第三方 UI `原生矩形绘制`，避免凭空依赖未知 组件 API。

## 技能栏 Tooltip

- 左侧技能栏保留 7 个竖向槽位。
- 鼠标悬停已学技能槽时显示竖向 tooltip：
  - 技能名
  - 所属功法
  - 属性
  - 消耗
  - 是否可释放
- 新增客户端 `ClientTechniqueData.TechniqueSummary`，从内置 cultivation JSON 中读取技能名、来源、属性，并暂以估算消耗显示；真正冷却/消耗/释放仍留给后续服务端技能系统校验。

## 打坐吐纳 HUD

- 新增 `BreathingHudOverlay`，打坐中独立显示在屏幕下方。
- HUD 显示：
  - 当前吐纳效率
  - 灵气浓度
  - 功法倍率
  - 灵根亲和
  - 5 秒结算进度条
- `SyncCultivationDataPacket` 追加同步当前位置灵气浓度与灵气性质，供客户端 HUD 使用。
