# 2026-06-09 0.1.27 UI/图形稳定性修复

## 修复内容

- `ClientEvents` 的 `ScreenEvent.Init.Post` 入口改为只匹配原版 `InventoryScreen.class`，避免 `CultivationStatsScreen` 继承背包后被重复注入“修仙”按钮。
- 新增客户端修仙/技能同步状态：登录、退出、重生、进入世界时重置；收到同步包后标记已同步。
- 打坐按键本地状态改为 pending，由服务端同步快照确认，避免本地静态真值和服务端快照双轨冲突。
- 左侧技能栏固定 7 槽，未同步时稳定显示 7 个空槽；已同步后保留服务端发送顺序，不再客户端按名称排序。
- 吐纳 HUD 未同步时稳定隐藏，pending 打坐状态下允许显示占位数据，避免依赖先打开背包。
- 修仙页保留 0.1.23 设计的基础/灵根/功法/负面状态四分页，并补充未同步占位提示、屏幕边界避让和原版槽位避让。
- 旧第三方 UI 风格皮肤绘制技能占位图标时使用单次纹理实例，避免 `setFillColor` 修改共享纹理造成颜色状态污染。

## 验证

- 已执行 `./gradlew build` 并通过。
- 产物：`build/libs/seeking_immortals-0.1.27.jar`。

## 仍需人工游戏内验证

- 打开原版背包只出现一个“修仙”入口，进入修仙页后不再重复出现入口按钮。
- 登录、重登、死亡重生、切维度/进入世界后，技能栏先稳定显示 7 空槽，收到同步后按服务端顺序显示技能。
- 未打坐时吐纳 HUD 不显示；按键进入/退出打坐与移动中断后，HUD 状态跟随服务端同步。
- 小屏幕、宽屏、GUI 缩放变化下，修仙页不遮挡原版槽位且不越界。
- 技能 tooltip、修仙页四分页、灵力条透明度没有污染后续原版 UI。
