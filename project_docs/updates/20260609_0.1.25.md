# 20260609 0.1.25：左侧技能栏 旧第三方 UI 槽位适配

## 本次实现

- 开始将客户端 UI 向 旧第三方 UI 迁移，本轮优先处理左侧技能栏。
- 左侧技能栏不再因为玩家没有已学技能而隐藏。
- 固定渲染 7 个技能槽位：
  - 有技能：显示 旧第三方 UI 绘制的槽位背景、占位图标、序号和技能首字母。
  - 无技能：显示透明背景空框和半透明序号。
- 新增 `ImmortalUiSkin` 客户端 UI 皮肤适配层，集中封装 旧第三方 UI 面板、技能槽和状态条纹理。
- 背包“修仙”页的主面板背景改为复用 旧第三方 UI 皮肤适配层。
- 在“修仙”页底部新增灵力/吐纳状态条，作为打坐吐纳状态栏迁移到 旧第三方 UI HUD 前的第一阶段可视化入口。
- 槽位绘制由原先分散在 Overlay 内的 旧第三方 UI 纹理字段，改为统一通过 `ImmortalUiSkin` 调用。
- 保留超过 7 个技能时的 `+N` 提示。

## 技术说明

- 当前仍通过 Forge `RenderGuiOverlayEvent.Post` 挂载 HUD，避免一次性重写所有客户端界面导致风险过高。
- 技能栏的槽位/背景已经先迁移为 旧第三方 UI 纹理渲染，后续可继续替换为完整 `组件树`/可交互 旧第三方 UI 组件。
- 所有修改仍位于 `client` 包和 `Dist.CLIENT` 事件订阅中，避免服务端加载客户端类。

## 待继续迁移

- 背包“修仙”页仍是原版 `Screen` + 原版按钮入口，后续迁移为 旧第三方 UI Screen/组件。
- 打坐吐纳状态栏目前仍主要依赖动作栏文字，后续迁移为 旧第三方 UI HUD/进度条。
- 功法/灵根信息界面仍待拆分为 旧第三方 UI 信息面板、悬浮详情和分页组件。
- 技能正式图标、悬浮详情、点击/快捷键释放、冷却、灵力消耗和服务端校验仍待实现。
