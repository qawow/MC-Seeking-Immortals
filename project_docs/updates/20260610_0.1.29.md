# 20260610 0.1.29 更新记录

## 修改内容

- 修复左侧技能栏 HUD 坐标防护：仍固定 7 槽，按 `screenHeight` 钳制顶部坐标，避免小屏或其他 UI 定位变更时跑到玩家界面右上角。
- 按用户选择的 UI 样式 B 方案，将修仙详情页改为独立全屏/居中大面板 `CultivationStatsScreen`。
- 原版背包只保留“修仙”入口按钮；点击后打开独立修仙 Screen。
- 独立面板保留四分页：基础、灵根、功法、负面状态。
- 面板宽高、分页按钮、状态条按当前 GUI Scale 与分辨率自适应。
- 返回逻辑：从背包入口进入时返回原版背包，其他入口关闭页面。
- 新增/补充修仙面板标题、关闭、返回背包语言键。

## 根因与修复说明

- 技能栏槽位本身应使用 `slotY = y + i * (SLOT_SIZE + SLOT_GAP)` 逐格下移，但旧实现没有根据 HUD 实际 `screenHeight` 做位置保护；当玩家界面/GUI 缩放和叠画面板定位混在一起排查时，容易表现为技能槽贴到不期望区域。0.1.29 保持 HUD 与 Screen 分离，并对左侧固定栏做高度钳制。
- 修仙页此前继承 `InventoryScreen` 并在背包旁边绘制面板，容易与原版背包槽位、JEI/XEI 和高 GUI Scale 相互挤压。0.1.29 改为独立 `Screen`，不再参与原版背包布局。

## 构建验证

- 首次 `./gradlew build` 失败：独立 `Screen` 中误保留 `InventoryScreen#renderTooltip` 调用。
- 移除该调用后重新执行 `./gradlew build` 成功。
- 升级版本到 `0.1.29` 后再次执行 `./gradlew build` 成功。
