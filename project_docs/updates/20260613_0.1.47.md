# 2026-06-13 0.1.47 飞剑/法宝飞行系统

## 新增内容

- 新增 `flying_sword` 青竹飞剑。
- 新增 `flying_artifact` 御风法宝。
- 新增 Curios `artifact` 槽，并允许飞剑/飞行法宝装备到该槽。
- 新增飞剑/飞行法宝模型、中文/英文语言和 16x16 占位贴图。
- 落地 1.1 六大核心属性的 Capability/NBT 存储与展示：`cultivation`、`mana`/`manaMax`、`divSense`、`bodyRef`、`qiDevRisk`、`tribRes`。
- 境界突破改为手动流程：修为达当前层/阶段上限后，通过按钮、按键或命令触发，服务端消耗破境丹/药力并结算成功率。
- 突破成功率加成改为加法模型：丹药 +5%~+20%、灵脉/灵眼 +15%、功法品质 +0%~+10%、连续失败执念每次 +5% 且最多 +30%。

## 飞行规则

- 筑基及以上玩家装备飞剑/法宝后，服务端授予飞行能力。
- 炼气期无法使用飞剑/法宝飞行。
- 不同境界有不同：
  - 每秒灵力消耗
  - 最大飞行高度
  - 飞行速度
  - 服务端水平/垂直速度钳制
- 灵力不足、卸下飞剑/法宝、境界不足或超过当前境界最大高度时，会撤销本模组授予的飞行。
- 撤销飞行时会恢复授予前的 `mayfly` 状态和飞行速度，避免覆盖创造/旁观或其他飞行来源。

## 突破流程规则

- 普通修炼、丹药和打坐吐纳只会把修为累积到当前层/阶段上限，不再自动跨层。
- 玩家可通过修仙面板“突破”按钮、默认未绑定的“尝试突破”按键或 `/seeking_immortals breakthrough` 发起突破。
- 服务端统一校验突破资源：已服用的突破辅助药力，或背包中 1 枚 `breakthrough_pill`。
- 成功时推进一个层/阶段/境界，并把新阶段修为进度清零；现有境界派生属性会随新境界生效。
- 失败时当前阶段修为进度回退到 80%，走火风险 +10%；风险达到 70% 以上会进行心魔检定。

## 突破成功率加成

- 丹药加成：普通破境丹和下品筑基丹为 +5%，`PillQuality` 预留中品/上品/极品为 +10%/+15%/+20%。
- 地灵之眼加成：当前复用灵气系统的灵脉核心判定，`AuraInfo.leyline()` 为 true 时 +15%。
- 功法品质加成：`TechniqueEntry` 可读取可选 `quality` 字段（0~10），缺省时按功法来源/ID 启发式估算，最高 +10%。
- 执念加成：连续突破失败每次 +5%，最多 +30%，突破成功后清零。
- 修仙面板和 `/seeking_immortals realm` 会显示当前上下文的突破概率；突破成功/失败反馈会显示丹药、灵眼、功法和执念加成概要。

## 核心属性规则

- 新玩家初始 `mana=100`、`divSense=5`，旧存档缺少对应字段时使用相同默认值。
- `cultivation` 与旧 `CultivationExp` 兼容，继续由当前修为累计值推导境界和小阶段。
- `manaMax` 仍由当前境界和小阶段计算，避免旧存档保存的上限与现行规则不一致。
- `bodyRef` 不设硬上限，`qiDevRisk` 限制 0~100%，`tribRes` 限制 0~90%。
- 修仙属性面板和 `/seeking_immortals qi` 会显示六大核心属性；网络同步包沿用既有字段顺序，未新增 packet 字段。
