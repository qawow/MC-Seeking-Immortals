# 2026-06-11 更新：0.1.34 技能槽持久化、绑定同步与冷却

## 版本

- `mod_version`：`0.1.33` -> `0.1.34`。

## 实现内容

- 玩家修炼数据新增并持久化 7 个技能槽绑定：
  - NBT 字段：`TechniqueSlots`。
  - 每槽保存 `techniqueId` 或空字符串。
  - 旧存档没有 `TechniqueSlots` 时按已学技能排序填入前 7 槽。
- 技能冷却进入玩家数据与同步链路：
  - NBT 字段：`TechniqueCooldownUntilTicks`。
  - 当前默认释放成功后 5 秒冷却。
  - 同步包下发每个技能剩余冷却 tick，客户端按本地 tick 展示剩余时间。
- `SyncLearnedTechniquesPacket` 同步内容扩展为：
  - 已学技能列表。
  - 7 个绑定槽位。
  - 冷却剩余 tick。
- `SetTechniqueSlotPacket` 用于保存/清空槽位绑定：
  - 校验 slot 必须为 0-6。
  - 非空 techniqueId 必须是玩家已学技能。
  - 传空字符串可清空槽位。
- `ReleaseTechniquePacket` 改为按绑定槽释放：
  - 校验槽位、空槽、已学、灵力和冷却。
  - 成功后消耗灵力、设置 5 秒冷却，并同步修炼数据和技能数据。
  - 冷却中提示剩余秒数。
- 客户端技能栏与快捷键改为读取绑定槽：
  - 左侧技能栏显示绑定槽位而不是直接按 learned 排序。
  - Z/X/C/G/B/N/M 快捷键按绑定槽位发送释放请求。
  - tooltip 增加冷却剩余秒数。
- 技能编辑界面保留原生最小能力：
  - 左键点击槽位：绑定同序号已学技能。
  - 右键点击槽位：清空槽位。

## 第二阶段补充

- 技能编辑界面补齐拖拽绑定：从右侧已学技能列表按住左键拖到左侧 7 个槽位，释放后发送 `SetTechniqueSlotPacket` 保存绑定。
- 槽位支持右键清空；保留左键槽位绑定同序号已学技能的兼容操作。
- 拖拽过程中显示当前技能名与“释放到左侧槽位”的提示。
- 7 个技能释放键与技能编辑键默认改为未绑定，避免 Z/X/C/G/B/N/M/R 硬冲突；保留“寻仙问道”按键分类和语言条目。
- 按键处理在任意 Screen 打开时会消耗/清理技能相关点击，不触发释放；技能编辑界面打开键也避免重复开屏。

- 修炼进度条改为先画背景/边框，再按 `tickCount % 100 / 100` 裁剪左侧填充区域绘制。
