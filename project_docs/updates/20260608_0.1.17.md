# 20260608_0.1.17 更新记录

## 本次目标

- 将右侧技能栏显示数量从 10 格调整为 7 格。
- 传承卷轴/书籍 Tooltip 改为按 cultivation JSON 文件内容动态展示同源条件与同源术法。
- 传承卷轴/书籍占位图标按传承来源分类，不再全部共用同一个占位图标。
- 灵根测试石改为单个只能使用 5 次，不可补充，用完只能重新合成。
- 在原版背包中新增“修仙”页签，用于显示本模组相关玩家数值。

## 已实现

1. `TechniqueSkillBarOverlay`
   - `MAX_VISIBLE_SKILLS` 从 10 改为 7。
   - 超出 7 个已学术法时继续显示 `+N` 提示。

2. `TechniqueDataManager` / `TechniqueManualItem`
   - 新增内置 cultivation JSON classpath 摘要读取。
   - Tooltip 保持四行：
     - `传承来源：xxx`
     - `使用条件：...`：按同 source 的 `attribute` 字段去重聚合。
     - `学会术法：...`：按同 source 的 `name` 字段聚合，最多展示 6 个并以“等 N 种”截断。
     - `右键使用`

3. 传承占位图标
   - 为 60 个传承来源生成分类占位纹理。
   - 为每个 `technique_manual_*` 生成/覆盖对应 item model，按 source 指向不同占位纹理。

4. `LingGenTestStoneItem`
   - 移除耐久消耗与灵石替代消耗逻辑。
   - 改为 NBT `RemainingUses`，默认 5 次。
   - 每次成功检测后消耗 1 次；剩余 0 次时物品破碎并播放效果。
   - Tooltip 显示剩余次数和不可补充说明。
   - 灵根检测后同步目标玩家修仙数据到客户端。

5. 修仙数值页
   - 新增 `SyncCultivationDataPacket`，同步境界、阶段、灵力、修为、年龄/寿元、灵根、属性、纯度、觉醒、特殊体质、已学术法数量、打坐与负面状态摘要。
   - 新增 `ClientCultivationData` 客户端缓存。
   - 新增 `CultivationStatsScreen`。
   - 原版背包界面新增“修仙”按钮，点击打开数值面板。
