# 20260609 0.1.19 更新日志

## 本次完成

- 新增统一术法/技能属性亲和计算器 `TechniqueAffinityCalculator`。
  - 支持直接传入 `TechniqueDataManager.TechniqueEntry` 或 `attribute` 字符串表达式计算亲和结果。
  - 支持中文灵根属性解析：金、木、水、火、土、风、雷、冰、暗、隐雷、隐暗、仙等。
  - 支持多段属性表达式，例如 `火/雷/隐雷`、`土/金/冰`、`水/冰`；首个识别到的灵根属性作为主属性，后续作为副属性。
  - 未识别的功法标签（剑法、魔道、血道、神魂等）不会导致崩溃，只是不提供灵根亲和加成，便于后续数据驱动扩展。
- `PlayerCultivation` 原有的术法亲和倍率、持续时间加成、效果等级加成接口已保留，但内部统一委托到 `TechniqueAffinityCalculator`，避免多处维护公式。
- `TechniqueDataManager` 新增统一亲和薄封装：
  - `calculateAffinity(PlayerCultivation, TechniqueEntry)`
  - `calculateAffinity(PlayerCultivation, String)`
  - `getAffinityMultiplier(PlayerCultivation, TechniqueEntry)`
  - `getAffinityMultiplier(PlayerCultivation, String)`
- `TechniqueDataManager#matchesAttributeCondition` 补充 `神念`、`身法`、`阵法` 等中性/通用条件，避免身法类功法被误判为必须具备某种灵根。
- 火弹符、金甲符、疾行符已改为首批统一亲和接口接入案例：
  - 火弹符使用 `火/雷/隐雷` 计算速度与投射物伤害倍率。
  - 金甲符使用 `土/金/冰` 计算抗性持续时间和效果等级。
  - 疾行符使用 `风/雷/隐雷` 计算速度持续时间和效果等级。
- 模组版本号更新为 `0.1.19`。
